指定されたファイルまたは直近の変更のコードを簡素化してください。

## 対象特定
```bash
git diff --name-only HEAD~1 2>/dev/null || git diff --name-only
```

## 簡素化の基準

### 削除候補
- 未使用のimport/変数/関数
- 不要なコメント（コードで自明な内容）
- 過剰な抽象化（1箇所でしか使われないヘルパー）
- デッドコード（到達不能なパス）

### リファクタリング候補
- ネスト3段以上 → 早期リターンで平坦化
- 重複コード → DRYに（ただし過剰抽象化しない）
- マジックナンバー → 定数化
- 長い関数（30行以上） → 責務分割

### やらないこと
- 動作を変える変更
- テストを壊す変更
- スペックに記載がない機能追加
- 不要なライブラリ追加

## 検証
簡素化後に必ず:
```bash
./scripts/run-tests.sh --unit
```
テストが全パスすることを確認してから完了とする。
