以下の手順でコミット→プッシュ→PR作成を一括実行してください。

## コンテキスト収集
まず現在の状態を確認:
```bash
git status
git diff --stat
git log --oneline -3
```

## 手順
1. 変更内容を分析し、適切なコミットメッセージを作成
2. 関連ファイルをステージング（機密ファイルを除外）
3. コミット
4. リモートにプッシュ
5. PRを作成（テンプレートに従う）

## PR作成ルール
- 種別（Spec/Task/Code/Fix/Docs）を明記
- 対応するスペックへのリンクを含める
- テスト結果を記載
- `scripts/validate-spec.sh` の実行結果を含める

## 注意
- mainへの直接プッシュは禁止
- PRのタイトルは70文字以内
- コミットメッセージは変更の「Why」を書く
