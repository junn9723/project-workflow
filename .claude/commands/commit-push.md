以下の手順でコミット→mainにプッシュを実行してください。

## コンテキスト収集
まず現在の状態を確認:
```bash
git status
git diff --stat
git log --oneline -3
```

## 手順
1. 変更内容を分析し、適切なコミットメッセージを作成
2. 関連ファイルをステージング（機密ファイルを除外）
3. コミット（メッセージは変更の「Why」を書く）
4. mainにプッシュ

## 注意
- `/verify-app` を実行済みであることを確認してからプッシュする
- 機密ファイル（.env, credentials等）をステージングしない
- コミットメッセージは変更の「Why」を書く
