PRレビューや作業中に発見した学びをCLAUDE.mdに反映してください。

## 収集する情報

### 直近のPRやコミットから
```bash
git log --oneline -10
```

### 現在のCLAUDE.md
CLAUDE.md の「ミスログ・学習記録」セクションを確認。

## 追記する内容の種類

### ミスパターン
- Claudeがやりがちな失敗パターン
- 形式: `- [日付] [カテゴリ] 内容 → 対策`

### ベストプラクティス
- うまくいったパターンや手法
- プロジェクト固有のコーディング規約

### 注意事項
- 外部依存の既知の問題
- 環境固有の注意点

## ルール
- CLAUDE.mdは2.5kトークン以内に保つ（肥大化防止）
- 古い・解決済みのエントリは削除
- 一般的すぎる内容は書かない（プロジェクト固有のもののみ）
